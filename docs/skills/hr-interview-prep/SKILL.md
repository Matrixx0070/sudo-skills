When to use: preparing an interview loop for a role, standardizing questions across panelists, or designing a scorecard so decisions are evidence-based and comparable. Aim for structured, bias-resistant interviews that predict on-the-job performance.

METHOD
1. Derive competencies from the job: pick 4-6 must-have competencies (technical + behavioral) tied to real responsibilities, not generic traits.
2. Distribute across the loop: assign each interviewer 1-2 competencies so coverage is complete and non-redundant; name the interview type (screen, technical, behavioral, values, hiring-manager).
3. Write questions: for each competency, give behavioral (STAR-style "Tell me about a time…") and situational prompts, plus 2-3 probing follow-ups to reach depth.
4. Define the rubric: for each competency describe what a 1 (below), 3 (meets), and 5 (exceeds) answer looks like with concrete signals.
5. Build the scorecard: competency → rating scale → evidence notes field, ending in a recommendation (Strong Yes / Yes / No / Strong No) with required written justification.
6. Add guardrails: interviewer briefing, legally-off-limits topics to avoid, and a debrief structure that shares evidence before scores to reduce anchoring.

OUTPUT FORMAT
- Role + competency map (competency → interviewer → interview type)
- Question bank grouped by competency (behavioral + situational + follow-ups)
- Scoring rubric (1/3/5 anchors per competency)
- Scorecard template + recommendation scale
- Debrief agenda and bias reminders
