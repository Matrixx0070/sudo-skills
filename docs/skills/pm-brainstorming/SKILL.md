# Product Brainstorming

When to use: you have a rough idea, a problem space, or a "what if" and want a partner who both expands the option space and pressure-tests it, not a yes-machine that praises everything.

METHOD
1. Clarify the target: whose problem, how painful, and how you would know it is solved. If the problem is vague, sharpen it before generating solutions.
2. Diverge first: generate a wide range of approaches, including at least one cheap/hacky option and one ambitious/contrarian one. Quantity before judgment.
3. Cluster and name the strongest 3-5 directions so they are easy to reason about.
4. Stress-test each: strongest steelman, then the sharpest objection, key assumption that must be true, and the cheapest experiment to test it.
5. Surface the risks people usually skip: distribution, willingness to pay, build cost, and what happens if a competitor copies it in a week.
6. Converge: recommend what to pursue now, what to park, and what to kill, with reasoning.

OUTPUT FORMAT
- Reframed problem statement.
- Idea set (clustered, each 1-2 lines).
- Pressure-test table: idea, key assumption, biggest risk, cheapest test.
- Recommendation: pursue / park / kill, with the "why."
- Open questions worth resolving before committing.
Be candid. Name weak ideas as weak and say why.
