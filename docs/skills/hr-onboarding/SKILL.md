## When to use

Reach for this when a new hire has accepted and you need an onboarding plan spanning before day one through the first 90 days. Balance logistics (access, paperwork) with belonging (people, context) and clear ramp goals.

**Not for:** writing the offer (use hr-draft-offer), designing the interview loop (use hr-interview-prep), or running the first performance cycle (use hr-performance-review).

## Method

1. **Pre-boarding (before day 1)** — order equipment, provision accounts/access, send I-9/tax/benefits paperwork, a welcome message, and the day-1 agenda in advance.
2. **Day 1** — manager 1:1, verify workspace/tools, team intro, security and compliance basics, and a where-to-get-help map.
3. **First week** — role-expectations conversation, scheduled stakeholder intros, read-in on team goals/docs, one small starter task for momentum, and shadowing where relevant.
4. **First 30/60/90** — define ramp milestones: 30 (learn systems + relationships), 60 (own a piece of work), 90 (fully productive, measurable goals). Make each concrete and checkable.
5. **Assign owners** — name who owns each item (IT, HR, manager, buddy). Decision point: any unowned item is a gap — assign it or drop it before publishing.
6. **Build feedback loops** — schedule a week-1 and day-30 check-in. Decision point: if the day-30 check-in surfaces blockers, adjust the 60/90 goals rather than letting the hire fall behind silently.

## Example

New hire: Data Analyst, remote. Pre-boarding — laptop shipped to arrive day -3, Okta + Snowflake + Looker access filed (owner: IT), benefits enrollment link sent (owner: HR). Day 1 — 9:00 manager 1:1, 10:00 tooling check, 14:00 team standup intro. Week 1 starter task: reproduce last month's retention dashboard. 30-day milestone: independently answer ad-hoc data questions; success signal = ships 3 correct queries unaided. 60: owns the weekly metrics report. 90: proposes one new metric with a documented definition. Buddy check-in on day 3 and day 14; manager check-in week 1 and day 30.

## Pitfalls

- Front-loading paperwork and access but never scheduling the human intros — the hire feels isolated.
- 30/60/90 milestones written as activities ("attend meetings") instead of outcomes with a success signal.
- No named owner per item, so equipment or access slips and day 1 stalls.
- Skipping the early check-in, so blockers surface only at review time.

## Output format

```
Pre-boarding checklist: item → owner → due
Day-1 agenda (timed): time → activity → owner
First-week plan: day → focus → activity
30/60/90 ramp: milestone → goal → success signal
Check-in schedule: week-1 + day-30 (owner)
```
