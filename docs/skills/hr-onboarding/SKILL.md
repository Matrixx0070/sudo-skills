When to use: a new hire has accepted and you need a ready-to-run onboarding plan — before day one through the first 90 days. Balance logistics (access, paperwork) with belonging (people, context) and clear ramp goals.

METHOD
1. Pre-boarding (before day 1): equipment order, accounts/access provisioning, I-9/tax/benefits paperwork, welcome message, and a day-1 agenda sent in advance.
2. Day 1: manager 1:1, workspace/tools setup verification, team intro, security and compliance basics, and where-to-get-help map.
3. First week: role expectations conversation, meet key stakeholders (schedule intros), read-in on team goals/docs, a small starter task to build momentum, and shadowing where relevant.
4. First 30/60/90: define ramp milestones — 30 (learn systems + relationships), 60 (own a piece of work), 90 (fully productive, measurable goals). Make each milestone concrete and checkable.
5. Assign owners: name who is responsible for each item (IT, HR, manager, buddy) so nothing falls through.
6. Build feedback loops: schedule a week-1 and day-30 check-in to catch blockers early and confirm the hire feels set up to succeed.

OUTPUT FORMAT
- Pre-boarding checklist (item → owner → due)
- Day-1 agenda (timed)
- First-week plan (day-by-day)
- 30/60/90 ramp milestones (goal → success signal)
- Buddy/manager check-in schedule
