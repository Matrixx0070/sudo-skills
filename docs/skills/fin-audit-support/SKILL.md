# Audit Support

## When to use
Use this to support an internal or external audit of internal control over financial reporting (ICFR), to design a SOX 404 test of a key control, or to prepare workpapers an auditor can re-perform without asking you a single question.

## METHOD
1. **Scope the control.** Record the control ID, the financial-statement assertion it addresses (existence, completeness, accuracy, valuation, cutoff), the risk it mitigates, and whether it is preventive or detective.
2. **Determine the testing approach.** Choose inquiry, observation, inspection, or re-performance. Re-performance is strongest; inquiry alone is never sufficient.
3. **Set sample size by frequency.** Apply standard minimums: daily controls ~25, weekly ~5, monthly ~2, quarterly ~2, annual 1. Increase for high-risk or automated-then-changed controls.
4. **Select the sample.** Use random or systematic (interval) selection from the full population; document the population source, total count, and selection method so it is reproducible.
5. **Execute the test.** For each item, capture the attribute tested, the evidence inspected, and pass/fail with the specific reason.
6. **Evaluate exceptions.** Classify each as control deficiency, significant deficiency, or material weakness; assess whether it is isolated or systemic.
7. **Conclude.** State whether the control is operating effectively and note any compensating controls.

## OUTPUT FORMAT
- **Control summary:** ID, assertion, risk, type, frequency.
- **Testing approach and rationale.**
- **Population and sample:** source, total, size, selection method.
- **Test-of-detail table:** item, attribute, evidence, result, notes.
- **Exceptions:** description, severity, root cause.
- **Conclusion:** effective / deficient, with compensating controls.
- **Open items and follow-ups.**
