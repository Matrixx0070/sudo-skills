When to use this skill when you need more than a single email: an onboarding, nurture, re-engagement, or promotional sequence where timing, order, and branching determine whether it converts.

METHOD

1. Define the sequence goal and the trigger that enrolls a contact (signup, download, cart abandon, inactivity). State the target conversion event.
2. Map the audience state at entry and what belief or action must change by exit.
3. Plan the arc. Decide how many emails, and give each one job (welcome, educate, prove, handle objection, offer, urgency). Avoid asking for the sale before trust exists.
4. Write each email: subject line plus alternate, preview text, concise body, single primary CTA.
5. Set timing. Assign send delays between steps and best-guess send windows; keep cadence tighter early, looser later.
6. Add branching: define behavior-based paths (opened/clicked vs. not, converted vs. not) and what changes on each branch.
7. Define exit conditions (converted, unsubscribed, hard bounce, goal met) so contacts do not receive irrelevant mail.
8. Attach benchmarks so results can be judged.

OUTPUT FORMAT

- Sequence name, goal, trigger, and target event.
- Email table: # | Purpose | Delay | Subject | CTA.
- Full copy for each email (subject + alt, preview, body, CTA).
- Branching logic: condition then path.
- Exit conditions.
- Benchmarks: expected open, click, and conversion ranges, with the caveat that these are directional estimates.
