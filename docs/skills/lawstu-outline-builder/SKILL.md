## When to use

Reach for this mid-to-late semester when your notes and briefs need to collapse into one attack-ready outline per course. The skill scaffolds the doctrinal skeleton and prompts you to fill each rule, element, and exception from memory — the act of building *is* the studying.

**Not for:** downloading a finished outline to memorize (a commercial or upperclassman outline you did not build teaches far less and, if submitted or shared against policy, breaches integrity). This guides you to write *your* outline; it never hands you a completed one. Not for the semester schedule (see `lawstu-study-plan`) or flashcard drilling (see `lawstu-flashcards`).

## Method

1. **Set the spine** from the syllabus — major doctrines in the order taught. Decision point: organize by *how the exam calls it* (issue-spotting order), not by textbook chapter, when they differ.
2. **Draft each rule from memory**, then check against your briefs and notes. The skill flags thin or missing entries but makes you write them.
3. **Break rules into elements and exceptions**, with a one-line trigger fact for each ("when you see X, test Y").
4. **Add a case tag per rule** — the shortest cite that anchors it. No fact summaries; this is an attack outline, not a treatise.
5. **Build issue-spotting triggers.** For each doctrine, note the fact pattern that surfaces it. Decision point: if two doctrines share a trigger, add a disambiguation note.
6. **Compress to a one-page checklist** you could scan in the last five minutes.

## Example

Torts, negligence branch: Duty → Breach → Causation (actual/proximate) → Damages, each with element, trigger ("unforeseeable plaintiff → Palsgraf duty question"), and case tag. The proximate-cause row cross-links to the intervening-cause row so you never miss the follow-on issue.

## Pitfalls

- Copying the casebook — length is not mastery; an attack outline is ruthless.
- Rules with no trigger fact, so you know the law but cannot spot it.
- Omitting minority rules the professor emphasized.
- Building it the night before; the outline must marinate through practice.

## Output format

```
Course | organizing logic (exam-call order)
Per doctrine:
  Rule | Elements | Exceptions | Trigger fact | Case tag
Cross-links: [doctrines that co-occur]
One-page attack checklist:
```
