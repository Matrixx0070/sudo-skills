When to use: sizing an offer, resolving a pay-equity question, setting or auditing bands for a role, or deciding a merit/promotion increase. Ground every number in a source and the role's level, location, and function.

METHOD
1. Define the role precisely: title, job family, level (IC or manager), primary location/geo tier, and remote policy. These drive every comparison.
2. Gather benchmarks: cite market sources (survey data, offers, ranges) and note percentiles (25th/50th/75th). Flag when data is thin or self-reported.
3. Set the band: pick a target percentile per your comp philosophy, then define band min / midpoint / max. Compute compa-ratio = actual / midpoint.
4. Place the candidate: map experience and skills to a point in the band; call out if they land below min or above max and why.
5. Model total comp: break out base, target bonus (% and $), equity (grant value, vesting, strike/FMV if options), and benefits. Show year-1 vs. steady-state.
6. Check equity: compare against peers in the same level/geo; surface any gap that needs justification or correction.

State assumptions and confidence explicitly. Never present a single point estimate as fact when the data is a range.

OUTPUT FORMAT
- Role summary (title, level, location, source date)
- Market table: percentile — base — bonus — equity — total
- Recommended band: min / mid / max + target compa-ratio
- Placement recommendation with rationale
- Total-comp model (year-1 and steady-state)
- Risks / equity flags / assumptions
