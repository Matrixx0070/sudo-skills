## When to use

Reach for this when running a review cycle, coaching a manager to write a fair review, or designing calibration so ratings are consistent and defensible. Keep feedback specific, evidence-based, and tied to expectations for the person's level.

**Not for:** the new-hire ramp (use hr-onboarding), pay decisions that follow a rating (use hr-comp-analysis), or interview scoring (use hr-interview-prep).

## Method

1. **Set the frame** — state the review period, the competencies/goals assessed, and the rating scale with a clear definition for each level.
2. **Self-assessment** — prompt the employee on accomplishments (with impact), goal progress, strengths, growth areas, and support needed. Ask for evidence, not adjectives.
3. **Manager template** — per competency/goal, capture concrete examples, a rating with justification, and forward-looking development actions. Separate performance-in-role from potential.
4. **Gather multi-source input** — incorporate peer/stakeholder feedback where available. Decision point: attribute recurring themes, not one-off gossip, and discount recency or halo effects.
5. **Draft the rating** — map evidence to the scale for the person's level. Decision point: write the summary so it would hold up if read aloud to the employee — if it wouldn't, it needs more evidence or a softer claim.
6. **Calibrate** — compare ratings across a peer group, surface inconsistencies and bias patterns, adjust with documented rationale, and finalize the distribution.

## Example

Employee: Marketing Manager (L4), H1 cycle. Competency "cross-functional influence." Evidence: drove the product-launch content plan across 3 teams, but two stakeholders cited late handoffs. Self-assessment claims "excellent collaboration"; manager rates 3/5 (Meets) with the handoff examples and a development action: run a shared launch calendar next half. Calibration: a peer L4 rated 4/5 on the same competency had cleaner delivery signals — the difference is documented, so the distribution holds up under review.

## Pitfalls

- Ratings built on adjectives ("great teammate") with no observable example behind them.
- Recency bias — weighting the last month over the full period.
- Conflating potential with performance, inflating a rating for someone "on the rise."
- Skipping calibration, so two managers apply the same scale differently and ratings aren't comparable.

## Output format

```
Review frame: period | competencies/goals | rating-scale definitions
Self-assessment template: prompts (accomplishments, goals, strengths, growth, support)
Manager review (per item): examples | rating | rationale | development action
Calibration guide: comparison method | bias checks | adjustment log
Final summary structure + delivery tips
```
