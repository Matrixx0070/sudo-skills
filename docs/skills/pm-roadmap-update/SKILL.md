# Roadmap Update

When to use: new information has arrived (metrics, customer asks, competitive moves, capacity changes) and your Now/Next/Later roadmap needs to reflect reality instead of last quarter's assumptions.

METHOD
1. Restate the current strategic goals and the roadmap's existing state so changes are traceable.
2. Intake new signals: what changed since the last update (data, requests, dependencies, constraints)? List each with its source.
3. Re-score candidate items on impact toward goals, confidence, effort, and time-sensitivity. Prefer a lightweight, consistent scheme (e.g., RICE or impact/effort) over gut feel.
4. Re-sort into Now (committed, in progress), Next (next cycle, sequenced), and Later (validated intent, not scheduled). Keep Now small enough to actually finish.
5. Make tradeoffs explicit: for every item promoted, name what was demoted or dropped and why.
6. Note dependencies, risks, and what would trigger another re-prioritization.

OUTPUT FORMAT
- Change summary: what moved and the reason.
- Now / Next / Later lists, each item with one-line rationale and rough size.
- Cut/deferred list with justification.
- Dependencies and risks.
- Decisions needed from stakeholders, with owners.
Distinguish committed items from aspirational ones; never present Later as a promise.
