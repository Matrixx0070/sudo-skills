# Variance Analysis

## When to use
Use this to explain why actuals differ from budget, forecast, or prior period — for a P&L line, a department, or a KPI — and to turn a raw dollar gap into named, quantified drivers leadership can act on.

## METHOD
1. **Frame the comparison.** State the metric, the actual, the baseline (budget / forecast / prior), and the total variance in dollars and percent; note favorable or unfavorable.
2. **Set materiality.** Define the threshold (dollar and/or percent) below which variances are not decomposed.
3. **Decompose into drivers.** Split the variance using the right bridge: price × volume for revenue, rate × hours for labor, volume × mix for product lines, plus one-time/timing items. Each driver must sum back to the total.
4. **Quantify each driver.** Compute the isolated impact of each, holding others constant.
5. **Attribute causes.** For each driver, give the business reason (new pricing, demand shift, headcount ramp, timing of spend).
6. **Build the waterfall.** Bridge baseline → each driver (up/down bars) → actual, so the decomposition is visible and ties out.
7. **Write the narrative and flag risk.** Call out drivers that will persist vs one-time.

## OUTPUT FORMAT
- **Metric / actual / baseline / total variance ($, %, F or U).**
- **Materiality threshold.**
- **Driver table:** driver, $ impact, % of total, cause, persistent?
- **Waterfall:** baseline → drivers → actual (label each step; confirm it ties).
- **Narrative:** plain-language explanation.
- **Risks / watch items and recommended actions.**
