When to use this skill when you need to know why a page or site underperforms in search and what to fix first, covering the four layers that move rankings.

METHOD

1. Set scope and intent. Confirm the URL(s), the target audience, and the primary search intent (informational, commercial, navigational, transactional). If unknown, infer from the content and state it.
2. Keywords. Identify the primary keyword, secondary and long-tail variants, and search intent match. Flag keyword cannibalization or missing target terms.
3. On-page. Check title tag, meta description, H1 and heading hierarchy, keyword placement, internal links, image alt text, URL structure, and readability. Note each issue.
4. Content gaps. Compare coverage against what ranking pages address; identify missing subtopics, thin sections, outdated facts, and missing schema/FAQ opportunities.
5. Technical. Review indexability (robots, canonical, sitemap), page speed signals, mobile-friendliness, structured data, and broken links, at the level the available information allows; note what needs a live crawl to confirm.
6. Prioritize. Score each finding by impact and effort and order the fixes.

OUTPUT FORMAT

- Scope, target keyword, and search intent.
- Findings by layer: Keywords, On-page, Content gaps, Technical, each as a short list.
- Issue table: Area | Issue | Impact (H/M/L) | Effort (H/M/L) | Fix.
- Prioritized action plan: top items first, quick wins flagged.
- What could not be verified without a live crawl or analytics access.
